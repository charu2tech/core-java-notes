1) What is object? <br>
   a) An object is a software bundle of related state and behavior. <br> 
   b) Software objects consist of state and related behavior. An object stores its state in fields (variables in some programming languages) and exposes its behavior through methods (functions in some programming languages). <br>
   c) Hiding internal state and requiring all interaction to be performed through an object's methods is known as data encapsulation. <br>
   d) Benifits of an object are modularity, information-hiding, code re-use, pluggability. <br>
